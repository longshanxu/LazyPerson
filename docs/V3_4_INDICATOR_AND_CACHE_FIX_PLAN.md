# V3.4 指标同屏与日线数据修正方案

## 背景

V3.3 已把页面调整为走势图优先，并只保留 MACD 和 LON 指标。但实际使用中仍有几个影响判断的问题：

- LON 缺少柱状图表达，不方便看 0 轴上方和下方强弱。
- MACD 和 LON 仍需要更直接地同屏观察，不适合再用选项卡切换。
- 自动画线在图右侧的价格标签会干扰阅读，只需要左侧百分比标签。
- 近 90 天应按自然日窗口过滤，但窗口内只展示真实交易 K，不能出现周末休市数据。
- 日线默认接口可能读到旧区间缓存，导致最新走势图数据不对。

## 实现范围

### 指标区

- MACD 和 LON 同屏纵向展示，不再使用选项卡。
- MACD 保留 DIF、DEA 线，并增加 MACD 柱状图。
- LON 保留 LONMA 线，并将 LON 主值改为柱状图。
- 两个指标的柱状图统一规则：
  - 0 轴上方为红色。
  - 0 轴下方为绿色。

### 日线主图

- 自动画线的右侧价格轴标签关闭。
- 图内只保留左侧百分比标签。
- `+20%`、`+50%`、`+80%` 继续作为重点线加粗标记。
- 水平线和趋势线保持实线。

### 90 自然日交易窗口

- 前端以最新日 K 日期为结束点，回推 90 个自然日。
- 只展示窗口内真实交易 K。
- 过滤周六、周日和 OHLC 缺失的记录。
- 指标数组按过滤后的 K 线索引同步截断。

### 缓存修正

- K 线 parquet 文件路径加入缓存 key 哈希，避免同一股票不同 `start/end` 区间互相覆盖。
- 默认日线缓存增加最新日期校验，缓存最新 K 线明显过旧时自动绕过并刷新。
- 默认日线请求缩短到满足指标计算的近一年原料，降低数据源刷新耗时。
- 默认日线优先使用 efinance/东方财富，显式历史区间仍按 AKShare、BaoStock、efinance 顺序兜底。

## 验收标准

- `002138` 默认日线最新日期应为当前可用交易日。
- 近 90 自然日窗口内不包含周六、周日记录。
- 页面同时显示 MACD 和 LON 两个指标图。
- 页面不显示 RSI 和资金流指标入口。
- MACD 和 LON 柱状图按 0 轴红绿分色。
- 图内左侧线位标签只显示百分比，不显示价格。
- `npm run test` 和 `npm run build` 通过。
