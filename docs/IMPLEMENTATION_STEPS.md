# 落地步骤

## 阶段 0：项目初始化

目标：建立可运行的前后端骨架和基础工程规范。

任务：

1. 初始化后端目录 `backend/`。
2. 初始化前端目录 `frontend/`。
3. 增加 `.env.example`，配置缓存目录、刷新间隔、默认数据源。
4. 增加基础启动脚本，例如 `make dev` 或 PowerShell 脚本。
5. 增加最小健康检查接口：`GET /api/health`。

验收：

- 本地能启动后端。
- 本地能启动前端。
- 前端能请求 `/api/health` 并显示服务状态。

## 阶段 1：数据接入与标准化

目标：先打通单只股票的行情链路。

任务：

1. 实现 `EFinanceAdapter`：
   - 实时行情。
   - 分钟 K 线。
   - 东方财富相关字段标准化。
2. 实现 `AKShareAdapter`：
   - 历史日 K。
   - 股票基础信息。
3. 预留 `BaoStockAdapter`：
   - 登录、登出封装。
   - 历史数据补齐。
4. 定义统一行情模型：
   - `StockQuote`
   - `KlineBar`
   - `MoneyFlow`
   - `DataQuality`

验收：

- 输入 `600519` 能拿到日 K。
- 输入 `600519` 能拿到分钟 K。
- API 返回字段不暴露第三方中文列名。

## 阶段 2：本地缓存

目标：同一数据不会每次都请求远端，离线时能查看已缓存数据。

任务：

1. 建立 SQLite 表：
   - `symbols`
   - `watchlist`
   - `cache_index`
   - `refresh_jobs`
2. 建立 Parquet 目录：
   - `data/cache/kline/day/{symbol}.parquet`
   - `data/cache/kline/minute/{period}/{symbol}.parquet`
   - `data/cache/money_flow/{symbol}.parquet`
3. 实现缓存策略：
   - 历史日线按交易日增量更新。
   - 分钟线按分钟级 TTL 更新。
   - 实时行情按秒级 TTL 更新。
4. API 增加 `refresh=true` 参数强制刷新。

验收：

- 第二次查询同一股票明显更快。
- 断网或数据源失败时，已缓存日线仍可展示。
- API 返回 `from_cache` 和 `source`。

## 阶段 3：指标计算

目标：前端可以直接拿到主流指标，不重复实现复杂计算。

任务：

1. 实现 MA：5、10、20、60。
2. 实现 EMA。
3. 实现 MACD：DIF、DEA、MACD histogram。
4. 实现 RSI：6、12、24。
5. 支持指标参数由前端传入。
6. 指标和 K 线按时间戳对齐。

验收：

- 日 K 接口可以附带 MA、MACD、RSI。
- 分钟 K 接口至少支持 MA 和 MACD。
- 指标缺失值处理稳定，不导致图表断裂异常。

## 阶段 4：前端 MVP

目标：完成一个真正能用的看盘界面。

任务：

1. 自选股看板：
   - 最新价。
   - 涨跌幅。
   - 成交额。
   - 换手率。
   - 更新时间。
2. 股票详情：
   - 日 K / 5 分钟 / 15 分钟 / 30 分钟 / 60 分钟切换。
   - K 线主图。
   - 成交量副图。
   - MACD / RSI 切换。
3. 搜索与添加自选。
4. 数据加载、失败、缓存兜底状态。

验收：

- 打开首页即可看到自选股看板。
- 点击股票能进入详情页。
- 切换周期不刷新整页。
- 数据源失败时前端显示缓存状态。

## 阶段 5：资金流与增强看板

目标：把走势和资金行为放在同一视图里。

任务：

1. 接入个股资金流。
2. 展示主力净流入、超大单、大单、中单、小单。
3. 看板增加资金流排序。
4. 增加日内资金流趋势图。

验收：

- 详情页能看到资金流曲线。
- 自选股可按资金净流入排序。
- 资金流接口有缓存和更新时间。

## 阶段 6：稳定性与体验

目标：让工具适合日常使用。

任务：

1. 增加数据源失败重试和降级。
2. 增加缓存清理命令。
3. 增加数据质量提示。
4. 增加常用指数：上证、深证、创业板、北证。
5. 增加导出 CSV。

验收：

- 数据源异常不会导致页面白屏。
- 用户能手动刷新、清理缓存、导出当前数据。
- README 有完整启动说明。
