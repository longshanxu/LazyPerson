# V3.2 行情终端 UI 与自然日窗口调整方案

## 背景

V3.1 已完成自动画线、颜色缓存和按钮控制图表，但当前界面仍偏通用管理后台，不够像主流看盘软件。同时“近 90 天”应按自然日计算，包含周末和节假日，而不是最近 90 根交易日 K 线。

## 调整目标

1. 日 K 展示窗口改为近 90 个自然日。
2. 自动画线基于近 90 个自然日内的交易日 K 线。
3. 前端请求日 K 时多取历史数据作为原料，再按最新日 K 日期回推 90 个自然日过滤。
4. K 线、MA、MACD、RSI 等指标数组随过滤窗口同步截断，避免指标与 K 线错位。
5. UI 改为主流行情终端风格：
   - 深色底色。
   - 黑色/深灰图表区。
   - 红涨绿跌。
   - 高密度行情信息。
   - 面板边框更硬朗。
   - 自选股列表更像行情列表。
   - 右侧个股摘要更像盘口/资料面板。
6. 图表主题同步改深色，网格线、坐标轴、成交量和均线颜色适配深色背景。
7. ECharts 指标区同步改深色主题。

## 自然日 90 天规则

以最新一根日 K 的日期为结束日，向前回推 90 个自然日作为窗口起点：

```text
windowStart = latestDailyKDate - 90 calendar days
```

只保留 `time >= windowStart` 的日 K。周末和节假日没有交易 K 线，但它们仍然计入 90 天窗口长度。

## 数据请求规则

- 日 K 请求从 90 根交易日提高到 140 根交易日作为原料。
- 前端收到数据后再按自然日窗口过滤。
- 分钟线仍保留原逻辑，不做自然日 90 天过滤。

## 验收标准

- `002138` 日 K 默认展示的是近 90 个自然日内的交易 K 线。
- 自动画线面板中的高点、低点、线位都基于近 90 个自然日窗口。
- 图表顶部文案明确显示“近90自然日”。
- K 线图、指标图、自选股和右侧摘要均为行情终端风格。
- `npm run test` 通过。
- `npm run build` 通过。
- 浏览器验证 `http://127.0.0.1:5175/` 可加载，无控制台错误。
