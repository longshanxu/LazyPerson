# V3.5 局域网访问与趋势通道调整方案

## 背景

当前开发服务只能在本机访问，局域网设备无法打开页面。同时自动画线的趋势线需要从局部波段线改为更直观的大范围双平行通道。

## 实现范围

### 局域网访问

- 前端 Vite 开发服务从 `127.0.0.1` 调整为 `0.0.0.0`。
- 后端 FastAPI 开发服务从 `127.0.0.1` 调整为 `0.0.0.0`。
- 保留前端 `/api` 代理到本机后端 `127.0.0.1:8008`，局域网设备访问前端时仍由 Vite 转发 API。
- README 增加局域网访问地址说明。

### 线位颜色

- `+20%` 固定为红色。
- `+50%` 固定为蓝色。
- `+80%` 固定为白色。
- 其他百分比线默认使用绿色。
- 线位颜色缓存版本升级，避免旧本地缓存覆盖新默认规则。

### 趋势通道

- 向上趋势改为双平行线：
  - 向上通道下轨。
  - 向上通道上轨。
- 向下趋势改为双平行线：
  - 向下通道上轨。
  - 向下通道下轨。
- 通道线覆盖近 90 自然日交易窗口的大范围，而不是只连接局部短波段。

## 验收标准

- 本机可继续访问 `http://127.0.0.1:5175/`。
- 局域网设备可访问 `http://<本机局域网IP>:5175/`。
- 线位标签颜色符合红、蓝、白、绿规则。
- 自动画线详情显示 4 条趋势通道线。
- `npm run test` 和 `npm run build` 通过。
