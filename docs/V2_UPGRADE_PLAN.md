# V2 升级方案

## 背景

V1 已完成从 0 到 1 的可运行 MVP，包括 FastAPI 后端、AKShare / efinance / BaoStock 数据源适配、本地缓存、技术指标和 React 前端工作台。

但 V1 更像技术验证页，存在明显问题：

- 页面视觉和交互较简陋，不像日常使用的分析工具。
- 自选股、搜索、周期切换和错误提示不够顺手。
- K 线图只满足基础展示，缺少成交量副图、悬浮信息和指标交互。
- 数据源失败时前端提示偏技术化。
- 默认 K 线返回数据量过大，前端负载和接口响应不够轻。

V2 的目标是把当前 MVP 升级为一个更可用、更稳定、更像工具的 A 股走势分析工作台。

## V2 目标

1. 页面从 Demo 感升级为专业、清晰、可日常使用的看盘工作台。
2. 自选股、搜索、周期切换、刷新和指标切换要流畅。
3. 数据源失败时，优先返回缓存或降级数据，并显示人能读懂的状态。
4. 图表能力升级为主图 + 成交量 + 指标 + 资金流，而不是单一 K 线。
5. 后端接口默认更轻，避免一次返回过大的历史数据。

## 产品形态

V2 采用三栏工作台布局。

```text
顶部状态栏
- 市场状态
- 当前时间
- 后端状态
- 数据源状态
- 全局刷新
- 缓存状态

左侧自选股栏
- 搜索输入
- 自选分组
- 最新价
- 涨跌幅
- 成交额
- 资金流简标
- 排序入口

中间主图区
- K 线主图
- MA 均线
- 成交量副图
- MACD / RSI / 资金流 tab
- 周期切换

右侧信息面板
- 个股摘要
- 开高低收

- 成交额

```

## 必做功能

### 1. UI 重做

目标：

- 首页直接进入可用工作台。
- 布局紧凑但不拥挤。
- 视觉风格偏工具型，不做营销页、插画页或装饰型页面。
- 涨跌颜色、表格密度、按钮状态和加载状态统一。

交付：

- 顶部状态栏。
- 左侧自选股栏。
- 中间图表区。
- 右侧个股信息区。
- 移动端基本可用，不要求完整交易终端体验。

### 2. 自选股看板增强

目标：

- 自选股不只是列表，要能作为快速切换入口。
- 数据源失败时仍能展示已有缓存和明确提示。

功能：

- 添加 / 删除自选股。
- 点击自选股切换详情。
- 默认自选股种子。
- 显示最新价、涨跌幅、成交额、更新时间。
- 支持按涨跌幅、成交额、资金流和自定义排序。
- 空状态和加载状态。

### 3. 搜索体验优化

目标：

- 搜索输入更像股票联想框。
- 添加后立即进入详情。

功能：

- 支持代码、名称搜索。
- 搜索结果显示代码、名称、市场。
- 添加后自动选中。
- 搜索失败时显示中文提示，不暴露底层异常。

### 4. K 线图增强

目标：

- K 线成为主工作区，而不是简单图表占位。

功能：

- 日 K、1m、5m、15m、30m、60m 周期切换。
- 主图展示 K 线和 MA。
- 成交量副图。
- 鼠标悬浮展示 OHLC、涨跌幅、成交量和指标值。
- 图例展示 MA 参数。
- 周期切换不白屏、不残留旧图层。

### 5. 指标面板增强

目标：

- 指标从底部文字升级为可切换副图。

功能：

- MACD tab。
- RSI tab。
- 资金流 tab。
- 指标参数预留配置入口。
- 指标缺失值要平滑处理，不导致图表异常。

### 6. 数据状态优化

目标：

- 数据不可用时，用户知道发生了什么，以及当前看到的数据是否可靠。

功能：

- 实时源失败时显示“实时行情不可用，使用最新日 K 兜底”。
- 缓存过期时显示“缓存数据，可能滞后”。
- 数据源超时显示“数据源超时，可稍后刷新”。
- 不直接显示 `JSONDecodeError`、`ConnectionError`、`TypeError` 等内部异常。

### 7. 后端接口优化

目标：

- 默认接口响应更轻，更适合前端图表。

改动：

- `GET /api/kline/{symbol}` 增加 `limit` 参数。
- 日 K 默认最近 300 条。
- 分钟线默认最近 1000 条。
- `quotes/realtime` 继续保留日 K 兜底。
- 统一错误响应结构。
- 数据质量字段增加业务化说明。

建议响应质量字段：

```json
{
  "source": "efinance",
  "from_cache": false,
  "updated_at": "2026-06-11T04:30:00Z",
  "stale": false,
  "fallback": false,
  "message": "实时行情"
}
```

## 实施顺序

1. 后端接口先升级：`limit`、错误消息、降级消息、默认自选。
2. 前端组件拆分：把当前页面拆成状态栏、自选栏、图表区、信息区。
3. UI 重做：完成三栏工作台。
4. 图表增强：成交量、指标 tab、悬浮信息。
5. 自选股和搜索增强。
6. 稳定性验证：测试、构建、浏览器检查。

## 验收标准

- 打开页面第一眼像一个可用的 A 股分析工具。
- 默认能看到自选股、价格、涨跌幅、K 线和成交量。
- 点击自选股能切换详情。
- 添加自选股后立即选中。
- 日 K、5m、15m 等周期切换不白屏、不崩溃。
- 数据源失败时能看到中文业务提示。
- 实时源失败时可使用最新日 K 兜底。
- `npm run test` 通过。
- `npm --prefix frontend run build` 通过。
- 浏览器验证 `http://127.0.0.1:5173` 页面可用。

## V2 不做

- 不做交易下单。
- 不做荐股。
- 不做策略回测。
- 不做云端同步。
- 不做登录和多用户权限。
- 不做行业板块、复杂因子和新闻舆情。

这些可以作为 V3 或独立专题规划。
