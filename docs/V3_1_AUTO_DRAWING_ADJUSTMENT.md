# V3.1 自动画线调整方案

> V3.2 已将“90 天”口径从“90 根交易日 K 线”修正为“近 90 个自然日窗口”。本文件保留 V3.1 原始方案记录，当前实现以 `docs/V3_2_MARKET_TERMINAL_UI_PLAN.md` 为准。

## 背景

V3 已完成基于日 K 的自动画线能力，但仍需要进一步贴近日常看盘习惯：

- 默认展示窗口应更短，聚焦近 90 天。
- 水平线应以 90 天最低点为起点，而不是只画固定 5 条目标线。
- 每条水平线应有独立颜色，且用户可以修改并保存。
- 90 天内还需要识别多个波段趋势线，用上升线和下降线包裹日 K。
- 图表不能被鼠标滚轮误缩放或误滚动，需要改为按钮控制。

## 调整目标

1. 日 K 默认只展示最近 90 根。
2. 自动画线只基于最近 90 根日 K。
3. 以 90 日最低点和最高点作为自动画线区间。
4. 从最低点开始画水平线：
   - 最低点本身画一根。
   - 之后每上涨 10% 画一根。
   - 一直画到不超过 90 日最高点。
5. 每条水平线使用独立颜色。
6. `+20%` 线默认使用红色。
7. 用户可以在右侧面板修改每条水平线颜色。
8. 颜色配置保存到浏览器本地缓存。
9. 自动识别两根上升趋势线和两根下降趋势线。
10. 禁止鼠标滚轮直接缩放或滚动图表。
11. 提供按钮控制：
    - 放大。
    - 缩小。
    - 左移。
    - 右移。
    - 全览。

## 水平线规则

设：

- `low` 为最近 90 根日 K 的最低价。
- `high` 为最近 90 根日 K 的最高价。

水平线从 `low` 开始：

```text
0%   = low
10%  = low * 1.10
20%  = low * 1.20
30%  = low * 1.30
...
```

只保留不超过 `high` 的线位。

如果最高点涨幅刚好不能覆盖太多 10% 台阶，至少显示最低点和最高点的区间信息。

## 颜色规则

默认颜色：

| 线位 | 默认颜色 |
| --- | --- |
| `0%` | 灰色 |
| `+10%` | 蓝色 |
| `+20%` | 红色 |
| `+30%` | 紫色 |
| `+40%` | 橙色 |
| `+50%` | 青色 |

用户修改颜色后，保存到浏览器 `localStorage`。

## 趋势线规则

在最近 90 根日 K 中寻找局部波段点：

- 局部低点：当前最低价低于或等于前后若干根 K 线最低价。
- 局部高点：当前最高价高于或等于前后若干根 K 线最高价。

绘制：

- 两根上升趋势线：连接最近的两个低点波段组合，用于观察支撑。
- 两根下降趋势线：连接最近的两个高点波段组合，用于观察压力。

如果局部点不足，使用分段高低点进行兜底，确保尽量生成趋势线。

## 图表交互规则

禁用：

- 鼠标滚轮缩放。
- 鼠标滚轮横向滚动。
- 鼠标拖拽平移。
- 触控缩放。

启用按钮：

- `+`：放大。
- `-`：缩小。
- `←`：左移。
- `→`：右移。
- `全览`：恢复完整 90 日视图。

## 验收标准

- `002138` 日 K 默认显示近 90 根。
- 自动画线基于近 90 根数据。
- 右侧面板显示 90 日最高点和最低点。
- 水平线从最低点开始，包含 `0%` 线。
- 每 10% 生成一根水平线，直到不超过最高点。
- `+20%` 默认红色。
- 修改颜色后刷新页面仍保留配置。
- 图表上显示两根上升趋势线和两根下降趋势线。
- 鼠标滚轮不改变图表缩放或位置。
- 图表按钮可以放大、缩小、左移、右移、全览。
- `npm run test` 通过。
- `npm --prefix frontend run build` 通过。
